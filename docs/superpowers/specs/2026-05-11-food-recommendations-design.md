# 美食饭店推荐模块设计

## 目标

新增一个“美食饭店推荐”模块，用来维护想吃清单和已去过饭店记录。第一版以手动录入和 GitHub 存储为主，不接大众点评 API，但保留大众点评链接字段，后续可以扩展“从链接导入”能力。

## 页面入口

- 新增页面路径：`/foods`
- 导航中新增入口：`美食饭店`
- 页面风格沿用现有“我的设备”和“推荐分享”的列表、卡片、编辑按钮、私钥保存模式。

## 数据模型

数据存储在 `src/app/foods/list.json`。

### 想吃清单

每条记录包含：

- `id`：稳定唯一 ID
- `name`：想吃的名称，可以是菜品名或饭店名
- `type`：`dish` 或 `restaurant`
- `city`：城市
- `area`：地区或商圈
- `tags`：标签列表
- `priority`：优先级，`low` / `medium` / `high`
- `note`：备注
- `dianpingUrl`：大众点评链接，可为空
- `createdAt`：创建时间
- `done`：是否已经完成

### 已去过饭店

每家饭店包含：

- `id`：稳定唯一 ID
- `name`：饭店名称
- `city`：城市
- `area`：地区或商圈
- `address`：详细地址
- `dianpingUrl`：大众点评链接，可为空
- `cover`：主图，可为空
- `cuisine`：菜系
- `pricePerPerson`：人均价格，可为空
- `rating`：整体推荐分，1 到 5 星
- `visitedAt`：到访日期
- `tags`：标签列表
- `note`：整体备注
- `recommendedDishes`：推荐菜列表
- `avoidDishes`：踩雷菜列表

菜品记录包含：

- `id`：稳定唯一 ID
- `name`：菜品名称
- `rating`：1 到 5 星
- `tags`：标签列表
- `note`：备注

## 标签体系

标签不做强制枚举，先用自由标签，页面上提供常用标签快速选择。

初始常用标签：

- 菜系：火锅、烧烤、日料、韩餐、川菜、粤菜、湘菜、甜品、咖啡、面包、快餐
- 场景：聚餐、约会、工作餐、宵夜、外卖、排队久、适合拍照、性价比高
- 口味：偏辣、清淡、重口、甜口、油腻、稳定、踩雷

## 页面结构

页面顶部是标题和搜索区：

- 标题：`美食饭店`
- 搜索框：支持输入店名、菜名、城市、地区、标签、链接关键词
- 类型筛选：全部、想吃、已去过
- 城市筛选：从数据中自动聚合
- 标签筛选：从数据中自动聚合
- 排序：推荐分数、最近去过、人均价格、星级最高

主体分两个视图：

- 想吃清单：更轻量，适合快速记录待探索目标
- 已去过饭店：卡片展示店名、城市地区、地址、人均、评分、标签、推荐菜、踩雷菜和大众点评链接

## 编辑能力

沿用现有私钥保存模式：

- 默认浏览模式
- 点击编辑进入编辑模式
- 支持新增想吃项
- 支持新增饭店
- 支持编辑和删除已有记录
- 点击保存时写入 `src/app/foods/list.json`
- 如果未认证，弹出私钥导入

第一版不做批量导入，不做大众点评自动抓取。

## 搜索和排序规则

搜索命中字段：

- 饭店名
- 想吃项名称
- 菜品名
- 城市
- 地区或商圈
- 详细地址
- 菜系
- 标签
- 备注
- 大众点评链接

排序规则：

- 推荐分数：已去过饭店按 `rating` 降序，想吃项排在后面
- 最近去过：已去过饭店按 `visitedAt` 降序，想吃项按 `createdAt` 降序
- 人均价格：有价格的饭店按价格升序，无价格项排后面
- 星级最高：饭店整体星级和菜品最高星级综合排序

## 后续扩展

后续可以增加：

- 从大众点评链接导入店名、地址、评分
- 店铺照片和菜品照片
- “再去意愿”字段
- 避雷指数
- 分城市统计
- 本地地图链接或导航链接

这些能力不进入第一版，避免阻塞基础模块上线。

## 验收标准

- `/foods` 页面可访问
- 导航可进入该页面
- 可查看想吃清单和已去过饭店
- 可按城市、地区、菜品、标签搜索
- 可按推荐分数排序
- 可新增、编辑、删除记录
- 保存后能通过现有 GitHub 写入模式更新 `src/app/foods/list.json`
- 构建通过
